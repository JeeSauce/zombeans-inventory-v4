| A-021 | 2026-07-11 | Weighted-average cost (`inventory_items.weighted_avg_cost`) is a **global, per-item** value (not per-branch). `post_purchase_receipt()` recomputes it by reading `inventory_balances.qty_on_hand`, which is branch-scoped; this is self-consistent for the single-branch MVP (Phase 2 seeds only one branch that holds raw ingredients). | Spec calls for a single weighted-average per item. ## Phase 4 additions

| ID    | Date       | Decision                                                                                                                                                                                                                                     | Rationale                                                                                                                                                                |
| ----- | ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| A-026 | 2026-07-13 | Recipes use three explicit kinds: `production` (one produced inventory output), `sale` (one product or variant deduction), and `modifier` (one inventory-affecting modifier-option deduction).                                               | Separates the recursively costed production graph from branch-sale and modifier deductions, and makes each target unique and unambiguous at the database layer.          |
| A-027 | 2026-07-13 | Planned cost is ingredients + consumable packaging + expected ingredient waste; unit cost divides that total by output quantity adjusted for expected yield. All result-boundary money arithmetic rounds to four decimal places.             | The requirements specify yield, waste, packaging, normalized base units, and `numeric(14,4)` without prescribing their exact arithmetic relationship.                    |
| A-028 | 2026-07-13 | Recursive costing follows only the active `production` recipe for a prepared input. A leaf without an active production recipe uses its current global `weighted_avg_cost`; a non-consumable container contributes zero.                     | Avoids ambiguity between production and sale graphs, preserves the Phase 3 valuation source, and implements the reusable-container rule.                                 |
| A-029 | 2026-07-13 | Activating a recipe version atomically retires the previous version, freezes the activated composition, and writes an append-only activation cost snapshot. Later weighted-average or supplier-price changes affect live recalculation only. | Preserves historical decision costs while still letting the Super Admin see current economics, satisfying critical scenario 8.                                           |
| A-030 | 2026-07-13 | Sale and modifier recipes may consume prepared sub-products, portioned products, packaging, and containers, but never raw ingredients or finished sellable items. Modifier recipes require `affects in ('inventory','both')`.                | Branch deductions should consume branch-held prepared components and packaging; raw-to-prepared conversion belongs in the production graph (Phase 4 side of scenario 9). |
